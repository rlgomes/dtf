Select

This child tag of query allows for you to define what you want to get back from the query. The fields specified in the select are the only ones you can then refer to in the where tag as well as the only fields available afterwards on the cursor. This is useful to avoid loading fields that may have binary blobs that would otherwise hurt the querying performance.



Author
Rodney Gomes


Children Tags

(field){0,*}

Usage Examples

Example #1
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<query event="hc.*" property="obj1" type="txt" uri="storage://OUTPUT/perf.db">
     <select>
         <field name="myfield"/>
     </select>
     <where>
         <eq field="myfield" value="${myvalue}"/>
     </where>
 </query>
