Filter
This tag is used to filter elements from an existing event file. You need to specify the srcuri and dsturi then with the select child tag you can specify which fields from the events to actually filter along and with the where child tag you can specify the exact conditions that have to be matched in order for the event to make the filter and be written to the dsturi.
Children Tags
select . where
Required Attributes
srcuri
The source URI to read the events from.
dsturi
The destination URI to write the events that have matched the where clause specified.
Usage Examples
Example #1
<filter dsturi="storage://OUTPUT/filtered.txt" event="write.event" srcuri="storage://OUTPUT/unfiltered.txt">
   <select>
      <field name="recordid" />
      <field name="hash" />
   </select>
   <where>
      <gt op1="5" op2="recordid" />
   </where>
</filter>
Example #2
<filter dsturi="storage://OUTPUT/filtered.txt" event="write.event" srcuri="storage://OUTPUT/unfiltered.txt">
   <where>
      <eq op1="123415123" op2="publish_date" />
   </where>
</filter>
