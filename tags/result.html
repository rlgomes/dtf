Result

Specifies location to record testcase results that are generated by the execution of testsuite and testscript tags.

Recording test results into other format is useful when you want to generate different reports of the same results. Also you'll find its easier to keep track of the history of running tests using this mechanism.



Author
Rodney Gomes


Children Tags

(){1,*}


Optional Attributes

uri

The output URI to use for the results types that may need to output to a file.

type

The currently supported result output types:

    Result Types
    Type Description
    xml Outputs the testsuite/testcase results in an XML format that can later be processed and analyzed to generate easier to read results from the test execution.
    console Outputs the testsuite/testcase results to the Logger so that the user can see all testsuite and testcase results directly in his logs.
    junit Outputs the testsuite/testcase results to the a format compatible with junit output. This is very useful for reusing tools that already took this type of input to generate reports.


Usage Examples

Example #1
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<result type="xml" uri="storage://OUTPUT/ut_results.xml">
     <testsuite name="unittests">
         <!-- Storage Tests -->
         <testscript uri="storage://INPUT/storage.xml"/>
     </testsuite>
 </result>

Example #2
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<result type="junit" uri="storage://OUTPUT/ut_results.xml">
     <testsuite name="unittests">
         <!-- Storage Tests -->
         <testscript uri="storage://INPUT/storage.xml"/>
     </testsuite>
 </result>
