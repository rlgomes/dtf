Jdbc

This tag can be used to talk to a database over JDBC, it currently allows you to do so for MYSQL databases using their 3.0 JDBC drivers. If you want to use other drivers for a different database make sure that those jars are in the classpath and that you specify the driver class and name to the tag before attempting to connect to your database.

Currently this tag has only be used to retrieve values and may not work as expected to issue create,insert or delete statements. In order to support those operations we may need to make some fixes.

Rodney Gomes


Events
jdbc Event
affectedrows

for update and delete returns the number of affected rows during that operation. For a select statement this attribute holds the number of return results. Be aware that the results are returned in the previously thrown events to this last jdbc event. So you should use a record tag inorder to record all previously thrown events. In other words when using a select you can get N events thrown being the last one this jdbc event.

columns

This only applies to each row returned in a select statement. This attribute would contain a comma separated list of the names of the columns returned during a select statement.

[column name]

This only applies to each row returned in a select statement. This attribute would contain each column mentioned in columns attribute will have its own attribute that can easily be referenced like so: ${jdbc.column_name}. This allows you to check each of the fields of the returned results from your select statement.



Usage Examples

Example #1
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<jdbc database="ydht" hostname="localhost" password="" port="3306" sql="DROP TABLE test" user="root"/>

Example #2
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<jdbc driverclass="oracle.jdbc.driver.OracleDriver" sql="SELECT * FROM refcount" url="jdbc:oracle:thin:asown/asown@masora104v.mail.re3.yahoo.com:1521/MASASV2"/>
