Eq

Evaluates the equality of the two operands that are identified by the attributes op1 and op2.



Author
Rodney Gomes


Optional Attributes

type

Identifies the type of the operands being evaluated. Supported DTF types:
Type Description
int Integer type
string String type

nullable

If nullable is set to true then if one of the operands is equal to null the condition will still evaluate to true. When set to false the behaviour of the comparison of the two operands is the normal conditional evaluation.



Required Attributes

op1

Identifies the first operand of a conditional tag.

op2

Identifies the second operand of a conditional tag.


Usage Examples

Example #1
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<if>
     <eq op1="${iteration}" op2="0"/>
     <then>
         <log>Iterations at 0!</log>
     </then>
 </if>
